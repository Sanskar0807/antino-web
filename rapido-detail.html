India's Largest Bike App

CASE REVIEW

Lona is an e-bike booking app which provides users to book e-bikes from the available nearby zones, shown inside the app. Currently there is a mobile number OTP system for user login/register. There is a wallet feature through which users can add money from a number of payment sources and then can use the amount for the rides.

CASE TIMELINE

2018 - 2019 September

TECHNOLOGIES USED

Java(Android native) | Razorpay payment gateway | Firebase messaging for push notifications | Retrofit for API

CHALLENGES

Implementing BLE protocols for communication between app and vehicle. Handling a large number of operations on a single activity for the Ride Activity. Securing the unauthorized communication of app with vehicle from multi login system Crashes because of version to version dependencies of Android & IOT Device sdk.

SOLUTION

We concluded to take help of the api call for turning On/off the device. Showing live location of all active devices in Dashboard. every time on opening the app the user data will be updated by getting the current info of the user form the API data. For handling the maps and ride operations together we used MVP design patterns and various code separation optimization techniques.
